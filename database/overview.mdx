# Database

InsForge provides a fully managed PostgreSQL database that automatically exposes REST APIs for all your tables through PostgREST. No backend code required.

## How It Works

When you create a table, it instantly becomes available as a REST API:

1. **Create a table** → `products`
2. **Get instant API** → `/api/database/records/products`
3. **Start using it** → Full CRUD operations immediately available

<Note>
The database is fully managed. You don't need to worry about connections, pooling, or infrastructure.
</Note>

## Quick Example

```bash
# Create a record (note: array syntax required)
curl -X POST http://localhost:7130/api/database/records/products \
  -H 'Authorization: Bearer YOUR_TOKEN' \
  -H 'Content-Type: application/json' \
  -H 'Prefer: return=representation' \
  -d '[{"name": "Widget", "price": 19.99}]'

# Query records
curl 'http://localhost:7130/api/database/records/products?price=gte.10' \
  -H 'Authorization: Bearer YOUR_TOKEN'
```

## Data Types

InsForge simplifies PostgreSQL's type system:

| Type | PostgreSQL Type | Description |
|------|----------------|-------------|
| `STRING` | `TEXT` | Unlimited text |
| `INTEGER` | `INTEGER` | Whole numbers |
| `FLOAT` | `DOUBLE PRECISION` | Decimal numbers |
| `BOOLEAN` | `BOOLEAN` | True/false |
| `UUID` | `UUID` | Unique identifiers |
| `DATETIME` | `TIMESTAMPTZ` | Date/time with timezone |
| `JSON` | `JSONB` | Structured data |

## System Tables

System tables are prefixed with underscore and have special rules:

| Table | Purpose | Access |
|-------|---------|--------|
| `_user` | User accounts | Read-only via database API |
| `_account` | OAuth provider accounts | System only |
| `_storage` | File metadata | System only |
| `_storage_buckets` | Bucket configuration | System only |
| `_config` | System configuration | System only |

<Warning>
The `_user` table is managed by the auth service. Use `/api/auth/*` endpoints for user modifications.
</Warning>

## MCP Integration

AI assistants interact with your database through InsForge's Model Context Protocol (MCP) server.

### Database Tools

| Tool | Purpose | Usage |
|------|---------|-------|
| `create-table` | Create new tables | AI designs schema based on your requirements |
| `modify-table` | Alter table structure | Add/drop/rename columns |
| `delete-table` | Remove tables | Permanently deletes table and data |
| `get-table-schema` | View table structure | Returns columns, types, relationships |

### How AI Uses MCP

When you ask your AI assistant to work with data:

1. **First run**: AI calls `get-instructions` to understand your backend
2. **Schema design**: Uses `create-table` with proper types and relationships
3. **Data operations**: Uses database API endpoints (via `get-db-api` documentation)
4. **Debugging**: Uses `debug-backend` if something fails

Example interaction:
```
You: "Create a products table for my e-commerce site"
AI: [Calls create-table with columns: id, name, price, stock, description]
AI: [Automatically adds created_at, updated_at timestamps]
AI: "Created products table. Ready to add products!"
```

The AI automatically handles:
- Proper data types (STRING, INTEGER, FLOAT, etc.)
- Foreign key relationships to `_user` table
- Index creation for performance
- Default values and constraints
